Network Routing Simulator

Milestone 4

This application simulates a network routing simulator with four routing algorithms: Random Routing Algorithm, Flooding Routing Algorithm, Breadth First Search Algorithm and Depth First Search Algorithm. Specifically, this application allows the user to create a network topology, step through simulations of these networks, and track metrics (packets transmitted and average hops) for the simulations.

Menu bar
> File
  1. Restore: opens a dialog which allows the user to select a network topology XML file to restore to the application
  2. Save: opens a dialog which allows the user to save the current network topology of the application to an XML file
  3. Exit: exits the application
> Simulation
  1. Step: steps the simulation, which forwards all messages at routers to their neighbours, and may generate a new message based on the message generation rate
  2. Undo: steps backwards in the simulation, which moves messages to their previous destination
  3. Message Generation Rate: opens a dialog which allows the user to set the rate at which messages are generated in the simulation
  4. Clear Messages: removes the messages from all routers and clears the metrics (note: you cannot step back after this)
  5. Routing Algorithm: opens a dialog which allows the user to select which routing algorithm to use for the simulation
> Help
  1. User manual: opens the current document
  2. About: team and version information
Control buttons
> Routers
  1. Add: opens a dialog which allows the user to enter a name for a new router, then lets the user click anywhere on the main view to add the router to the topology
  2. Remove: removes the selected router from the topology if a router was selected
> Connections
  1. Add: after clicked, allows the user to click and drag from one node another to create a new connection between routers on the topology
  2. Remove: removes the selected connection between routers from the topology if a connection was selected
Guide

To add a new router first click on the add button in routers, then user will then have a prompt to enter a name and then will be prompted to click on the screen where the router should be added. This can be repeated as many times as desired.

To remove a node, click on the node, and the border will change colour to notify you that it has been selected.

To create a new connection from one node to another click on one router and drag to another router.

To remove a connection, click on the connection and then click on the remove connection button.

To simulate the network, go to the 'Simulation' menu in the menu bar and click 'Step'. The Routing algorithm that was last selected in the RoutingAlgorithm dialog (or RandomRoutingAlgorithm as default) will be used to simulate a step in the network.

The undo functionality will only move back messages to their previous destinations and restore the metrics to their values from the last step. This will not undo adding/removing routers/connections. After restoring a network topology from an XML file, you cannot undo any stored messages/metrics that initially load (the previous messages/metrics do not get saved to XML).

After making a modification to a network topology (add/remove routers/connections) you will not be able to undo to a state earlier than the modification point. Additionally, after clearing all messages on the topology through the 'Clear Messages' menu button, you will not be able to undo to a state earlier than that point.
